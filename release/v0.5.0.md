# POKit Starter v0.5.0

## Summary

v0.5.0은 POKit의 issue workflow를 실제 운영 가능한 형태로 다듬은 릴리즈입니다. 핵심은 cross-runtime `pokit-issue` 계약, main-session/context-manager와 병렬 worker 실행 원칙, metrics 수집, sprint retrospective, 그리고 release gate의 deletion accounting 표준화입니다.

## What's New

- **Cross-runtime Issue Workflow Contract** (POK-105) — `pokit-issue`를 docs/specs, implementation/hotfix, cleanup/deletion, release/readiness gate 전반에 쓰는 Harness Issue workflow engine으로 정의했습니다.
- **Main Session Context Manager** (POK-106) — main session은 context, gate, state, integration만 관리하고 durable work는 task-type worker subagents가 수행하는 원칙을 표준화했습니다.
- **Layer A Metrics Collection** (POK-097) — issue execution metrics 12-field schema와 ignored run-log storage를 추가했습니다.
- **Sprint Retrospective Standard** (POK-098) — sprint close summary와 retrospective를 구분하고 v0.4.0 dogfooding retro를 남겼습니다.
- **Codex Model Efficiency Probe** (POK-107) — release audit, routine docs/spec checks, fast sidecar checks에 맞는 Codex worker model tier를 기록했습니다.
- **Net-Deletion Accounting Standard** (POK-108) — product surface, validation artifact, physical hygiene cleanup, generated/ignored run artifact, moved/integrated artifact 기준을 release gate evidence에 반영하도록 표준화했습니다.

## Changed

- **Project Structure Cleanup** (POK-100) — `node_modules`, lock/config leftovers, stale local release build leftovers, `.DS_Store`류 잔재를 정리했습니다.
- **Dead Reaper Cleanup** (POK-091) — stale benchmark 문서를 research benchmark 영역으로 이동하고 active references/tests를 정리했습니다.
- **Commit/Rebase Integration Gate** (POK-110) — v0.5.0 누적 작업을 clean commit/rebase/push 가능한 상태로 통합했습니다.
- **Starter Install Guidance** — README Quick Start와 starter metadata를 v0.5.0 archive 기준으로 갱신했습니다.
